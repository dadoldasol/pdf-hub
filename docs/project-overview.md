# 프로젝트 개요

## 1. 프로젝트 목표

PDF Knowledge Hub는 Android Camera HAL, kernel, ISP, chipset 관련 기술 PDF를 분석해 검색 가능한 지식허브로 만드는 시스템이다.

단순히 PDF 본문을 검색하는 것이 아니라, 문서 안의 기술 개념을 엔티티로 추출하고 요약/분류/관계 정보를 저장해 사용자가 연관 지식을 탐색할 수 있게 한다.

## 2. 해결하려는 문제

기술 PDF는 보통 다음 문제가 있다.

| 문제 | 설명 |
|---|---|
| 정보 분산 | IFE, BPS, CSID 같은 주제가 여러 PDF에 흩어져 있다. |
| 검색 어려움 | 파일명이나 키워드를 모르면 필요한 내용을 찾기 어렵다. |
| 문맥 부족 | 검색 결과만으로 기능, 한계, 구현 위치, 디버깅 방법을 함께 보기 어렵다. |
| 출처 추적 어려움 | 요약 내용을 원문 페이지와 연결하기 어렵다. |
| 지식 재사용 어려움 | 칩셋별 변경점, 코드 흐름, 디버깅 방법이 누적되지 않는다. |

## 3. 주요 사용자

| 사용자 | 필요 기능 |
|---|---|
| Camera HAL 개발자 | block 기능, call flow, 코드 위치 검색 |
| Kernel/driver 개발자 | driver 변경점, log keyword, debugging guide 검색 |
| ISP bring-up 담당자 | chipset별 지원 기능, 한계, known issue 확인 |
| 신규 합류 개발자 | 핵심 개념별 지식 카드로 빠르게 학습 |

## 4. 핵심 사용 시나리오

### 4.1 IFE 관련 지식 탐색

사용자가 `IFE`를 검색하면 시스템은 다음 정보를 묶어서 보여준다.

- IFE 개요
- 지원 기능
- 관련 ISP block
- 구현 파일 또는 함수
- 관련 chipset
- 디버깅 키워드
- 한계 또는 주의사항
- 출처 PDF와 페이지 번호

### 4.2 자연어 질문

예시 질문:

```text
SM8650에서 IFE 변경점과 디버깅 방법 알려줘.
```

시스템 응답은 다음 구조를 따른다.

- 요약 답변
- 관련 문서 조각
- 관련 엔티티
- 관련 PDF 페이지 링크
- 추가로 볼 만한 개념

### 4.3 원문 확인

검색 결과 또는 지식 카드에서 PDF 파일명과 페이지 번호를 클릭하면 원문 페이지를 확인할 수 있어야 한다.

## 5. 프로젝트 범위

### 포함 범위

- PDF 업로드
- 텍스트 추출
- 페이지/청크 저장
- 엔티티 추출
- 요약 생성
- 임베딩 생성
- 벡터 검색
- 지식 카드 생성
- 기본 그래프 관계 저장
- 원문 페이지 링크 제공

### 제외 또는 후순위

- 완전 자동 OCR 고도화
- 복잡한 표 구조 의미 해석
- Neo4j 기반 고급 그래프 추론
- 실시간 협업
- 사용자 권한 관리
- 대규모 운영 모니터링
- 다국어 번역 자동화

## 6. 성공 기준

MVP 성공 기준은 다음과 같다.

- PDF 10개 이상을 업로드해 처리할 수 있다.
- 각 PDF의 페이지 텍스트와 청크가 저장된다.
- `IFE`, `CSID`, `BPS`, `HAL`, `kernel`, `debug` 같은 키워드와 자연어 질문으로 검색할 수 있다.
- 검색 결과가 원문 PDF 페이지와 연결된다.
- 주요 엔티티별 지식 카드가 생성된다.
- 최소한의 관계 그래프를 API로 조회할 수 있다.
